Issues faced while coding and testing

 

 

1.   We were not able to decide input format as in some test cases nodes start from 0 and in some they start from 1.So we wrote code for both of them with some lines commented out so that we can act according to the test cases.

 

2.   There was a trade-off for space complexity with time complexity. So we implemented with both adjacency list and adjacency matrix. As adjacency matrix works faster for short test-cases but can’t process more than 10,000 nodes.

 

3.   We were not able to test on very large test cases above 20,000 nodes as they were out of machine capacity.

 

4.   In order to store path in residual graph earlier we were storing each edges which was making things complex but we followed dsu similar approach and made parent array which made things simpler.

 

5.   Apart from that we were able able to implement all tasks on our own with few debugging issues here and there.
